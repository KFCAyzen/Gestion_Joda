1. L'Application Joda Company

1.1 Présentation

L'application Joda Company est le système de gestion interne de l'agence Joda Company, spécialisée dans l'accompagnement des étudiants vers des bourses d'études en Chine. Elle permet de gérer l'intégralité du cycle de vie d'un dossier étudiant : de l'inscription jusqu'à l'obtention du visa.

3. Connexion et Navigation Commune

3.1 Connexion

Version Web (navigateur)

Étapes détaillées :
  1. Ouvrez un navigateur sécurisé (Chrome, Firefox, Edge, Safari)
  2. Allez à l'adresse officielle : https://joda.company
  3. Vérifiez que vous voyez le cadenas vert (connexion sécurisée)
  4. Saisissez votre nom d'utilisateur (format : InitialePrénom + Nom, ex : JDupont)
  5. Saisissez votre mot de passe de façon sécurisée (respectez la casse)
  6. Cliquez sur « Se connecter »
  7. Vous êtes redirigé automatiquement vers votre espace :
    • Étudiants : Portail étudiant (suivi de dossier)
    • Personnel (Agent, Superviseur, Admin) : Interface de gestion
  8. Votre nom et rôle s'affichent en haut à droite

Version Desktop (application installée)

Étapes détaillées :
  1. Double-cliquez sur l'icône Joda Company sur le bureau ou le menu Démarrage
  2. L'application démarre (première ouverture peut prendre 5-10 secondes)
  3. Une fenêtre dédiée s'ouvre avec le formulaire de connexion
  4. Saisissez votre nom d'utilisateur
  5. Saisissez votre mot de passe
  6. Cliquez sur « Se connecter »
  7. L'application synchronise les données avec le serveur cloud (affiche une barre de progression)
  8. Vous accédez à votre espace avec une copie locale des données (fonctionnement offline possible)
Note importante :
  • Étudiants : Vous êtes redirigés automatiquement vers le portail étudiant. C'est votre espace personnel pour suivre votre candidature et paiements.
  • Personnel de l'agence (Agent, Superviseur, Admin) : Vous accédez à l'interface de gestion pour traiter les dossiers étudiants.
  • Mode offline : L'application desktop continue de fonctionner sans Internet — les modifications seront synchronisées au retour de la connexion.

Problèmes de connexion courants

ProblèmeSolution
« Identifiants invalides »Vérifiez le nom d'utilisateur exact (respect de la casse). Cliquez sur « Mot de passe oublié » si nécessaire.
« Serveur indisponible »Vérifiez votre connexion Internet. Attendez quelques minutes. Réessayez.
« Page se rafraîchit mais ne se connecte pas »Videz le cache du navigateur. Essayez en mode navigation privée. Réessayez.
« Erreur de certificat SSL »NE CLIQUEZ PAS sur « continuer sans sécurité ». Contactez votre admin.

3.2 Première connexion — changement de mot de passe

Lors de la toute première connexion, une fenêtre de changement de mot de passe s'affiche. Ce changement est obligatoire avant tout accès à l'application.

Règles du mot de passe : minimum 8 caractères, majuscules + minuscules + chiffres recommandés.

3.3 Mode hors-ligne (application desktop)

L'application desktop fonctionne sans connexion Internet. Les données sont disponibles localement et synchronisées automatiquement dès le retour de la connexion. Certaines fonctionnalités (uploads, notifications temps réel) nécessitent une connexion active.

6. Portail Étudiant — Gestion des Documents

Étudiant

6.1 Documents acceptés

Types : Passeport, Relevés de notes, Diplômes, Photo d'identité, Acte de naissance, Lettre de motivation, Certificat médical, Preuve de paiement. Formats : PDF, JPG, PNG. Taille max : 10 MB par fichier.

6.2 Envoyer un document

Procédure détaillée :
  1. Depuis le menu, cliquez sur Documents
  2. Vous voyez la liste des documents attendus et leur statut
  3. Pour un document, cliquez sur « Choisir un fichier » (bouton bleu)
  4. Une fenêtre de sélection s'ouvre — naviguez vers votre fichier
  5. OU glissez-déposez directement le fichier dans la zone (plus rapide)
  6. Vérifiez que le fichier est correct et lisible (test rapide : double-cliquez pour l'ouvrir)
  7. Cliquez sur « Envoyer » (bouton vert)
  8. L'application valide le fichier :
    • Format accepté ? (PDF, JPG, PNG uniquement)
    • Taille ≤ 10 MB ?
    • Fichier complet (pas corrompu) ?
  9. Un message de succès s'affiche : « Document envoyé »
  10. Le statut du document passe à En attente (en cours d'examen par l'équipe)
  11. Optionnel : Cochez « Notifier l'équipe » avant envoi pour envoyer une alerte à votre agent
  12. Attendez la validation de l'équipe. Vous serez notifié si :
    • ✓ Document est accepté → statut Validé
    • ✗ Document est refusé → raison expliquée, nouvelle tentative possible

Points importants

  • Format : Acceptez PDF, JPG, PNG — pas de Word, Excel, ou autres formats
  • Taille : Chaque fichier max 10 MB — si plus volumineux, comprimez l'image d'abord
  • Qualité : Documents doivent être clairs et lisibles (pas de photos floues, pas de pages manquantes)
  • Date limite : Envoyez vos documents rapidement — plus vous attendez, plus votre dossier avance lentement
  • Documents manquants : Si vous recevez une notification « Document manquant », envoyez-le sans attendre

Erreurs possibles et solutions

ErreurCauseSolution
« Format de fichier non supporté »Fichier Word, Excel, ZIP, etc.Convertissez en PDF ou JPG/PNG. Utilisez un convertisseur en ligne (CloudConvert, Smallpdf).
« Fichier trop volumineux »Fichier > 10 MBComprimez l'image (réduction qualité) ou le PDF (logiciel gratuit : ILovePDF).
« Impossible d'envoyer »Connexion Internet perdue ou interruption uploadVérifiez votre connexion. Réessayez. Pas de perte — les fichiers ne montent que si succès total.
« Document refusé — raison indiquée »Qualité insuffisante, document incomplet, ou mauvais documentLisez la raison fournie. Corrigez-la. Envoyez une nouvelle version.

13.2 Enregistrer un paiement (onglet Encaisser)

Procédure détaillée :
  1. Depuis le module Frais, cliquez sur l'onglet « Encaisser »
  2. Sélectionner l'étudiant — tapez le nom/prénom pour filtrer la liste
  3. Sélectionner le type de paiement (obligatoire) :
    • Bourse (procédure) — Frais de candidature
    • Mandarin — Cours de langue chinoise
    • Anglais — Cours d'anglais
    • Inscription — Frais d'inscription administratifs
    • Autre — Paiements divers
  4. Sélectionner la tranche — selon le type (T1-T4 pour cours, T1 pour procédure)
  5. Saisir le montant — format numérique uniquement, en FCFA (ex : 1000000 pour 1 million)
  6. Saisir la date de paiement — utilisez le calendrier (ne peut pas être future)
  7. Optionnel : Ajouter une date limite de paiement
  8. Optionnel : Ajouter une note (ex : « Reçu en espèces »)
  9. Vérifiez tous les champs obligatoires (marqués d'une * rouge)
  10. Cliquez sur « Enregistrer »
  11. Validation serveur effectue les vérifications : montant > 0, dates cohérentes, pas de doublon
  12. Un message de succès s'affiche : « Paiement enregistré »
  13. Une écriture comptable (crédit) est créée automatiquement
  14. Le paiement apparaît immédiatement dans l'onglet « Validés »
  15. L'étudiant voit le paiement dans son portail

16. Comptabilité — Livre Comptable Complet

Admin / Super Admin — Gestion complète

16.1 Présentation

Le livre comptable consolide toutes les entrées (revenus) et sorties (dépenses). Les entrées sont créées automatiquement lors des validations de paiements. Les sorties sont saisies manuellement.

16.2 Créer une sortie (dépense)

  1. Comptabilité → « Nouvelle dépense »
  2. Catégorie : Loyer / Salaires / Fonctionnement / Matériels / Fournitures / Transports / Communication / Partenaires / Divers
  3. Montant, date, description
  4. Optionnel : joindre un justificatif (facture, bon de caisse)
  5. Enregistrer → apparaît en rouge « Non validée »

16.3 Valider une sortie

Trouver la sortie (fond rouge) → vérifier → cliquer « Valider » → marquée avec votre nom et date.

16.4 Exports (PDF, Excel/CSV, rapport HTML)

Trois formats d'export sont disponibles pour la période et la portée filtrées :

FormatUsage
PDFRapport officiel à archiver (en-tête Joda Company, tableaux, totaux, balance)
Excel / CSVAnalyse dans un tableur — toutes les opérations ligne par ligne
Rapport HTMLOuvre une fenêtre d'impression pour imprimer directement

Appliquer les filtres période/portée/catégorie → cliquer sur le bouton du format souhaité. PDF et CSV se téléchargent ; le HTML ouvre l'impression.

16.5 Budgets prévisionnels

L'onglet Budgets permet de définir un montant prévisionnel par catégorie et par période, puis de suivre les dépenses réelles par rapport à la prévision.

  1. Onglet Budgets → « Nouveau budget »
  2. Sélectionner la catégorie (ex : Loyer, Salaires), la période et le montant prévu
  3. Enregistrer
  4. Le module compare automatiquement : Montant prévu / Consommé / Reste disponible / % consommé (jauge verte < 80%, orange 80-100%, rouge > 100%)

16.6 Catégories personnalisées

L'onglet Catégories permet d'ajouter vos propres catégories d'entrées et de sorties au-delà des catégories par défaut.

  1. Onglet Catégories → « Nouvelle catégorie »
  2. Choisir le type (Entrée ou Sortie) et saisir le nom (ex : « Subventions », « Frais de visa »)
  3. Enregistrer → la catégorie devient disponible dans les listes de création d'entrées/sorties et dans les filtres

17. Gestion des Utilisateurs et Permissions

Admin / Super Admin

17.1 Créer un utilisateur

Procédure détaillée :
  1. Menu → Utilisateurs → cliquez sur « Nouvel utilisateur »
  2. Remplir les informations personnelles :
    • Prénom (obligatoire) — 2-50 caractères
    • Nom (obligatoire) — 2-50 caractères
    • Téléphone (optionnel) — Format +237 6XX XXX XXX
  3. Remplir les identifiants :
    • Nom d'utilisateur (unique, obligatoire) — Ex : JDupont (4-20 caractères, pas d'espaces)
    • Email (unique, obligatoire) — Ex : jean.dupont@joda.company
  4. Sélectionner le rôle (obligatoire) :
    • User — lecture basique
    • Agent — opérations courantes
    • Supervisor — validations + suppressions
    • Admin — gestion complète (sauf création admin)
    • Super Admin — accès total (réservé super_admin uniquement)
  5. Définir un mot de passe temporaire (obligatoire) :
    • Minimum 8 caractères
    • Au moins 1 majuscule, 1 minuscule, 1 chiffre, 1 caractère spécial (!@#$%)
    • Exemple : « JoDa#2026!Sec »
  6. Vérifiez tous les champs obligatoires
  7. Cliquez sur « Créer »
  8. Le compte est créé avec le flag automatique : « Doit changer le mot de passe »
  9. L'utilisateur sera forcé de changer son mot de passe à la première connexion
  10. Communiquez les identifiants de façon sécurisée (jamais par email non chiffré)
Hiérarchie des rôles : Un admin peut créer des utilisateurs de tous les rôles inférieurs (User, Agent, Supervisor). Seul un Super Admin peut créer des admins. Vous ne pouvez pas créer un rôle égal ou supérieur au vôtre.

17.3 Permissions granulaires

Au-delà des permissions standards du rôle, vous pouvez affiner les droits de chaque utilisateur pour des cas spéciaux. Exemple : un agent qui peut aussi valider les paiements, ou un superviseur qui n'a pas accès à la comptabilité.

Procédure

  1. Module Utilisateurs → trouvez l'utilisateur
  2. Cliquez sur « Gérer les permissions »
  3. Un panneau affiche les permissions par module
  4. Cochez/décochez les permissions selon vos besoins
  5. Les modifications sont sauvegardées automatiquement
  6. L'utilisateur doit se reconnecter pour que les nouveaux droits prennent effet

Cas d'usage courants

CasConfigurationRaison

Conseil : Les permissions granulaires sont puissantes mais complexes — gardez-les simples et documentées. La plupart des cas peuvent être gérés avec les rôles standard (User, Agent, Supervisor, Admin).

22. Workflows Métier Complets

Workflow 1 — Onboarding complet d'un étudiant
  1. [Agent] Créer la fiche étudiant (module Étudiants)
  2. [Agent] Communiquer les identifiants de connexion portail à l'étudiant
  3. [Agent] Créer la candidature (module Candidatures)
  4. [Agent] Si cours de langue : inscrire aux cours (Étudiants → vue détail → Cours)
  5. [Agent] Créer les paiements attendus (Frais → Encaisser)
  6. [Étudiant] Se connecte au portail, change son mot de passe
  7. [Étudiant] Soumet ses documents via le portail
  8. [Superviseur] Valide les documents dans le dossier
  9. [Agent] Met à jour le statut du dossier
  10. [Étudiant] Déclare ses paiements au fur et à mesure
  11. [Superviseur] Valide les paiements déclarés
  12. [Superviseur] Passe le statut à « Admission validée » puis « Visa en cours »
  13. [Agent] Clôture le dossier (statut « Terminé »)
Workflow 2 — Traitement d'un paiement
  1. Notification reçue : un étudiant a déclaré un paiement
  2. [Superviseur] Frais → onglet À valider
  3. Vérifier la preuve attachée (bouton « Voir la preuve »)
  4. Cliquer Valider si conforme → statut passe à « Payé »
  5. Écriture comptable créée automatiquement
  6. Reçu disponible pour téléchargement
Workflow 3 — Clôture mensuelle comptable
  1. [Admin] Comptabilité → filtrer sur le mois écoulé
  2. Valider toutes les sorties non validées
  3. Vérifier la cohérence des entrées avec les paiements
  4. Exporter PDF du rapport mensuel
  5. Archiver le rapport dans les dossiers de l'entreprise
Workflow 4 — Gestion d'un document non conforme
  1. Ouvrir le dossier de l'étudiant
  2. [Superviseur] Cliquer Rejeter sur le document problématique
  3. Saisir la raison du rejet (ex : « Image floue — rescanner »)
  4. L'étudiant est notifié et voit la raison dans son portail
  5. [Étudiant] Soumet un nouveau document
  6. [Superviseur] Valide le nouveau document

23. Sécurité et Bonnes Pratiques

  • Se déconnecter en quittant l'application sur un poste partagé

25. Communication — Messages & SMS

Agent / Superviseur / Admin

25.1 Présentation

Le module Communication permet d'envoyer des emails et des SMS aux étudiants, individuellement ou en masse. Deux onglets : Messages (email + SMS simultané) et SMS (SMS seul, avec solde de crédits).

25.2 Envoyer un message (Messages)

  1. Module Communication → onglet « Messages »
  2. Cocher un ou plusieurs étudiants (ou « Tout sélectionner »)
  3. Saisir le sujet et le corps du message
  4. Cliquer sur « Envoyer » → email HTML à chaque destinataire + SMS simultané si numéro disponible

25.3 Envoyer un SMS (SMS)

  1. Onglet « SMS » → consulter le solde de crédits affiché
  2. Sélectionner les destinataires (étudiants avec numéro)
  3. Saisir le texte (max 160 caractères — surveiller le compteur)
  4. Envoyer → normalisation automatique des numéros camerounais (237XXXXXXXXX)
Attention : Les envois de masse sont irréversibles. Vérifiez le contenu et les destinataires. Surveillez le solde SMS (consommable payant) et privilégiez l'email pour les messages longs.

26. Configuration des Frais

Admin / Super Admin

26.1 Présentation

Ce module définit les montants et tranches par défaut appliqués dans toute l'application. Quand un agent inscrit un étudiant à un cours ou crée une procédure, les montants proviennent de cette configuration. La modifier affecte les nouvelles opérations uniquement (les paiements existants ne changent pas).

26.2 Les 4 onglets

OngletConfigure
Bourse BachelorTranches/montants procédure bourse Licence
Bourse MasterTranches/montants procédure bourse Master
MandarinTranches/montants cours de mandarin
AnglaisTranches/montants cours d'anglais

26.3 Modifier la configuration

  1. Sélectionner l'onglet du service à configurer
  2. Modifier le label ou le montant d'une tranche, en ajouter ou en supprimer
  3. Le total se recalcule automatiquement
  4. Cliquer sur « Enregistrer » (ou « Réinitialiser » pour revenir aux valeurs par défaut)
Attention : Une erreur de montant se répercute sur toutes les nouvelles inscriptions. Vérifiez deux fois avant d'enregistrer, idéalement avec validation de la direction.
