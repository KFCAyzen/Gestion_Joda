1. L'Application Joda Company

1.1 Présentation

L'application Joda Company est le système de gestion interne de l'agence Joda Company, spécialisée dans l'accompagnement des étudiants vers des bourses d'études en Chine. Elle permet de gérer l'intégralité du cycle de vie d'un dossier étudiant : de l'inscription jusqu'à l'obtention du visa.

3. Connexion et Navigation Commune

3.1 Connexion

  1. Ouvrez l'application Joda Company (navigateur ou application desktop)
  2. Saisissez votre nom d'utilisateur
  3. Saisissez votre mot de passe
  4. Cliquez sur « Se connecter »
  5. Redirection automatique vers votre espace (portail étudiant ou interface de gestion)
Note : Les étudiants sont redirigés automatiquement vers le portail étudiant. Le personnel de l'agence accède à l'interface de gestion.

3.2 Première connexion — changement de mot de passe

Lors de la toute première connexion, une fenêtre de changement de mot de passe s'affiche. Ce changement est obligatoire avant tout accès à l'application.

Règles du mot de passe : minimum 8 caractères, majuscules + minuscules + chiffres recommandés.

3.3 Mode hors-ligne (application desktop)

L'application desktop fonctionne sans connexion Internet. Les données sont disponibles localement et synchronisées automatiquement dès le retour de la connexion. Certaines fonctionnalités (uploads, notifications temps réel) nécessitent une connexion active.

4. Portail Étudiant — Accès et Première Connexion

Étudiant — Ce chapitre vous concerne directement.

4.1 Accès au portail

Le portail étudiant est votre espace personnel en ligne pour suivre votre candidature. Vos identifiants (nom d'utilisateur + mot de passe temporaire) vous sont remis par votre agent Joda Company lors de votre inscription.

4.2 Tableau de bord étudiant

Après connexion, votre tableau de bord affiche :

  • Carte Statut dossier — statut actuel, cliquable pour les détails
  • Carte Paiements — montants dus et payés
  • Carte Documents — état de vos documents
  • Cloche notifications — badge rouge avec alertes non lues

4.3 Navigation du portail

  • Tableau de bord — vue d'accueil
  • Dossier — détails de votre candidature
  • Documents — envoi et suivi des pièces
  • Paiements — frais et déclarations
  • Profil — informations personnelles

16. Comptabilité — Livre Comptable Complet

Admin / Super Admin — Gestion complète

16.1 Présentation

Le livre comptable consolide toutes les entrées (revenus) et sorties (dépenses). Les entrées sont créées automatiquement lors des validations de paiements. Les sorties sont saisies manuellement.

16.2 Créer une sortie (dépense)

  1. Comptabilité → « Nouvelle dépense »
  2. Catégorie : Loyer / Salaires / Fonctionnement / Matériels / Fournitures / Transports / Communication / Partenaires / Divers
  3. Montant, date, description
  4. Optionnel : joindre un justificatif (facture, bon de caisse)
  5. Enregistrer → apparaît en rouge « Non validée »

16.3 Valider une sortie

Trouver la sortie (fond rouge) → vérifier → cliquer « Valider » → marquée avec votre nom et date.

16.4 Export PDF

Appliquer les filtres période/catégorie → cliquer « Exporter PDF » → rapport avec en-tête Joda Company, tableau entrées/sorties, totaux et balance.

17. Gestion des Utilisateurs et Permissions

Admin / Super Admin

17.1 Créer un utilisateur

  1. Menu → Utilisateurs → « Nouvel utilisateur »
  2. Remplir : prénom, nom, username (unique), email (unique), téléphone
  3. Rôle : agent / supervisor / user (admin peut créer jusqu'à supervisor)
  4. Mot de passe temporaire (min 8 caractères)
  5. Créer → flag « doit changer le mot de passe » activé
Règle : Un admin ne peut créer que des rôles inférieurs (jusqu'à supervisor). Seul un super_admin peut créer des admins.

22. Workflows Métier Complets

Workflow 1 — Onboarding complet d'un étudiant
  1. [Agent] Créer la fiche étudiant (module Étudiants)
  2. [Agent] Communiquer les identifiants de connexion portail à l'étudiant
  3. [Agent] Créer la candidature (module Candidatures)
  4. [Agent] Si cours de langue : inscrire aux cours (Étudiants → vue détail → Cours)
  5. [Agent] Créer les paiements attendus (Frais → Encaisser)
  6. [Étudiant] Se connecte au portail, change son mot de passe
  7. [Étudiant] Soumet ses documents via le portail
  8. [Superviseur] Valide les documents dans le dossier
  9. [Agent] Met à jour le statut du dossier
  10. [Étudiant] Déclare ses paiements au fur et à mesure
  11. [Superviseur] Valide les paiements déclarés
  12. [Superviseur] Passe le statut à « Admission validée » puis « Visa en cours »
  13. [Agent] Clôture le dossier (statut « Terminé »)
Workflow 2 — Traitement d'un paiement
  1. Notification reçue : un étudiant a déclaré un paiement
  2. [Superviseur] Frais → onglet À valider
  3. Vérifier la preuve attachée (bouton « Voir la preuve »)
  4. Cliquer Valider si conforme → statut passe à « Payé »
  5. Écriture comptable créée automatiquement
  6. Reçu disponible pour téléchargement
Workflow 3 — Clôture mensuelle comptable
  1. [Admin] Comptabilité → filtrer sur le mois écoulé
  2. Valider toutes les sorties non validées
  3. Vérifier la cohérence des entrées avec les paiements
  4. Exporter PDF du rapport mensuel
  5. Archiver le rapport dans les dossiers de l'entreprise
Workflow 4 — Gestion d'un document non conforme
  1. Ouvrir le dossier de l'étudiant
  2. [Superviseur] Cliquer Rejeter sur le document problématique
  3. Saisir la raison du rejet (ex : « Image floue — rescanner »)
  4. L'étudiant est notifié et voit la raison dans son portail
  5. [Étudiant] Soumet un nouveau document
  6. [Superviseur] Valide le nouveau document

23. Sécurité et Bonnes Pratiques

23.1 Gestion des accès

  • Ne jamais partager ses identifiants de connexion
  • Se déconnecter en quittant l'application sur un poste partagé
  • Changer son mot de passe régulièrement (recommandé : tous les 3 mois)
  • Signaler immédiatement à l'admin tout accès suspect

23.2 Données confidentielles

  • Les données étudiants sont personnelles et confidentielles (RGPD/loi locale)
  • Ne pas partager les exports comptables en dehors de la direction
  • Les logs d'activité restent à usage interne administration uniquement

23.3 En cas de problème

ProblèmeÀ contacter
Impossible de se connecterVotre responsable / admin
Bug ou comportement inattenduPrestataire technique Joda Company
Données incorrectesVotre admin pour correction via logs
Problème de stockage (limite atteinte)Admin + prestataire technique
Accès suspect détectéAdmin immédiatement (désactivation du compte)
