1. Introduction — Rôles et Permissions

1.1 Présentation

Ce manuel est destiné aux agents et superviseurs de Joda Company. Il couvre l'ensemble des opérations quotidiennes : gestion des étudiants, des candidatures, des dossiers, des paiements, et plus encore.

1.2 Différence entre Agent et Superviseur

Les permissions varient selon votre rôle. Les fonctionnalités marquées Superviseur ne sont accessibles qu'aux superviseurs et niveaux supérieurs.

ModuleAgentSuperviseur
Tableau de bordLectureLecture
ÉtudiantsVoir, Créer, ModifierVoir, Créer, Modifier, Supprimer
CandidaturesVoir, Créer, ModifierVoir, Créer, Modifier, Supprimer
DossiersVoir, Créer, ModifierVoir, Créer, Modifier, Supprimer, Valider
UniversitésLecture seuleVoir, Créer, Modifier
PaiementsVoir, Créer, ModifierVoir, Créer, Modifier, Valider
ComptabilitéNon accessibleConsultation
UtilisateursNon accessibleNon accessible
Logs d'activitéNon accessibleNon accessible
PerformancesLectureLecture

1.3 Hiérarchie des rôles

La hiérarchie complète dans l'application (du plus faible au plus fort) :

  1. Étudiant (student) — portail dédié uniquement
  2. Utilisateur (user) — accès lecture basique
  3. Agent — opérations courantes
  4. Superviseur — opérations + validations
  5. Admin — gestion complète
  6. Super Admin — accès total

2. Connexion et Navigation Générale

2.1 Connexion

Accès Web (navigateur)

Étapes de connexion — Version web :
  1. Ouvrez votre navigateur (Chrome, Firefox, Edge, Safari)
  2. Accédez à l'adresse officielle : https://joda.company/login
  3. Vérifiez que l'adresse commence par https:// (connexion sécurisée)
  4. Saisissez votre nom d'utilisateur (exemple : JDupont)
  5. Saisissez votre mot de passe (respect de la casse)
  6. Cliquez sur « Se connecter »
  7. L'application effectue une vérification du serveur (peut prendre 2-3 secondes)
  8. Vous arrivez sur le tableau de bord principal
  9. Votre nom et rôle (Agent/Superviseur) s'affichent en haut à droite

Accès Desktop (application installée)

Étapes de connexion — Version desktop :
  1. Double-cliquez sur l'icône Joda Company sur le bureau ou dans le menu Démarrage
  2. Attendez le chargement de l'application (première ouverture peut prendre 5-10 secondes)
  3. Une fenêtre dédiée s'ouvre avec le formulaire de connexion
  4. Saisissez votre nom d'utilisateur
  5. Saisissez votre mot de passe
  6. Cliquez sur « Se connecter »
  7. L'application se synchronise avec la base de données cloud (affiche une barre de progression)
  8. Vous accédez au tableau de bord avec vos données locales synchronisées
  9. Avantage offline : Si vous perdez la connexion Internet, vous continuez à travailler. Les modifications seront synchronisées automatiquement au retour de la connexion.

Tableau des problèmes courants et solutions

ProblèmeCause probableSolution
« Identifiants invalides »Nom d'utilisateur ou mot de passe incorrect (respect de la casse)Vérifiez le nom d'utilisateur exact (majuscules/minuscules). Cliquez sur « Mot de passe oublié » si nécessaire.
« Erreur de connexion — serveur indisponible »Serveur Joda Company en maintenance ou problème réseauAttendez quelques minutes. Vérifiez votre connexion Internet. Réessayez.
« Certificat SSL invalide » (navigateur)Problème de sécurité ou cache navigateur corrompuVidez le cache du navigateur. Essayez en mode navigation privée. Réessayez.

« Connexion réussie mais tableau de bord vide »Données non encore synchronisées (lenteur réseau)Attendez 10-15 secondes. Rafraîchissez la page (F5 ou Cmd+R).
Conseil : Pour les accès fréquents, l'application web est plus rapide. L'application desktop est idéale si vous devez travailler sans connexion Internet stable.

4. Gestion des Étudiants

4.1 Accès au module

Menu latéral → Étudiants

4.2 Vue liste des étudiants

La page affiche la liste de tous les étudiants avec :

  • Nom, prénom
  • Email de contact
  • Téléphone
  • Date d'inscription
  • Boutons d'actions

Utilisez la barre de recherche (par nom/prénom/email) ou le filtre par sexe pour retrouver rapidement un étudiant. La liste est paginée (20 étudiants par page).

4.3 Créer un nouvel étudiant

Étapes détaillées :
  1. Cliquez sur « Nouvel étudiant » (bouton rouge en haut à droite)
  2. Vous entrez dans le formulaire composé de plusieurs onglets ou sections.
  3. Onglet 1 — Identité personnelle :
    • Prénom (obligatoire) — 2-50 caractères, lettres uniquement
    • Nom (obligatoire) — 2-50 caractères, lettres uniquement
    • Email personnel (obligatoire) — Format valide (exemple@domaine.com). Doit être unique dans la base de données.
    • Téléphone (optionnel) — Format international accepté (ex : +237 6XX XXX XXX) ou format national (6XX XXX XXX)
    • Date de naissance (optionnel) — Sélectionnez via calendrier. Doit être une date révolue.
    • Sexe (optionnel) — Sélection : Masculin, Féminin, Autre
    • Nationalité (optionnel) — Liste déroulante des pays
  4. Onglet 2 — Profil académique :
    • Niveau d'études actuel (optionnel) — Licence, Master, Doctorat, Professionnel, Autre
    • Filière / Spécialité (optionnel) — Texte libre (ex : « Génie Informatique »)
    • Niveau de langue (optionnel) — HSK (pour mandarin), TOEFL/IELTS (pour anglais), score actuel
    • Diplôme obtenu (optionnel) — Baccalauréat, Licence, Master, etc.
  5. Onglet 3 — Programme d'études :
    • Type de programme (obligatoire) — Sélectionnez un parmi :
      • Procédure seule — Frais de candidature uniquement
      • Procédure + Cours — Candidature + cours de langue
      • Cours seuls — Cours de langue sans procédure
    • Note : Ce choix détermine les types de paiement à créer plus tard dans le module Frais.
  6. Vérifiez que tous les champs obligatoires (marqués d'une * rouge) sont remplis.
  7. Cliquez sur « Enregistrer » (bouton vert en bas du formulaire).
  8. L'application effectue une validation :
    • Vérification de l'unicité de l'email
    • Vérification du format du téléphone
    • Vérification que la date de naissance est antérieure à aujourd'hui
  9. Un compte étudiant est automatiquement créé avec :
    • Email système : prenom.nom@students.joda.app (pour accéder au portail étudiant)
    • Mot de passe temporaire : Chaîne aléatoire de 12 caractères (à communiquer en toute sécurité)
  10. Un email d'activation est envoyé automatiquement à l'adresse email personnelle de l'étudiant avec ses identifiants et un lien d'activation.
  11. Vous êtes redirigé vers la vue détails de l'étudiant nouvellement créé.

Erreurs courantes et solutions

Message d'erreurCauseSolution
« Email déjà utilisé »Cet email existe déjà dans la base de donnéesUtilisez un email différent ou vérifiez si l'étudiant existe déjà avec un alias email.
« Format de téléphone invalide »Numéro qui ne correspond pas aux formats acceptésUtiliser le format +237 XXXXXXXXX (avec code pays) ou 6XX XXX XXX (national).
« Champs obligatoires manquants »Au moins un champ marqué * n'est pas rempliVérifiez les onglets, complétez prénom et nom, sélectionnez un type de programme.
« Date de naissance invalide »Date future saisie ou format incorrectUtilisez le calendrier pour sélectionner une date. Elle doit être antérieure à aujourd'hui.
Important : Ne communiquez jamais les identifiants par email non sécurisé. Utilisez un canal sécurisé (SMS, discussion directe, portail interne) ou imprimer les identifiants en main propre.

Informations générées automatiquement à la création :

ChampValeur générée

6.5 Changer le statut d'un dossier

Transitions de statut autorisées

Tous les statuts ne peuvent pas être atteints depuis n'importe quel état. Le système applique des règles de transition :

Statut actuelStatuts possibles suivantsRôle requis
Documents reçusEn attente, En cours, Document manquantAgent
En attenteEn cours, Document manquant, En attente universitéAgent
En coursDocument manquant, En attente université, Admission validée, Admission rejetéeSuperviseur
Document manquantEn attente, En coursAgent
En attente universitéEn cours, Admission validée, Admission rejetéeSuperviseur
Admission validéeVisa en cours, TerminéSuperviseur
Admission rejetéeTerminéSuperviseur
Visa en coursAdmission validée, TerminéSuperviseur
TerminéAucun (statut final)—

Procédure détaillée

  1. Depuis la liste Dossiers, cliquez sur le dossier à modifier pour l'ouvrir
  2. Localisez la section « État du dossier » ou « Statut » (généralement en haut de la page)
  3. Observez le statut actuel (couleur badge indique l'état) et les statuts possibles affichés
  4. Cliquez sur le menu déroulant « Changer le statut »
  5. Seuls les statuts autorisés selon la transition actuelle apparaissent
  6. Sélectionnez le nouveau statut souhaité
  7. Si applicable, une zone de commentaire supplémentaire peut apparaître (exemple : « Raison du rejet » si admission rejetée)
  8. Remplissez les champs additionnels si demandés
  9. Cliquez sur « Enregistrer » (bouton vert)
  10. Un message de confirmation affiche le nouvel état
  11. La modification est enregistrée dans l'historique du dossier avec timestamp et nom de l'utilisateur
  12. L'étudiant reçoit automatiquement une notification du changement de statut dans son portail

Cas d'erreur et blocages

SituationBlocageRaison / Solution

6.6 Notes internes

Le champ « Notes internes » permet de laisser des informations pour l'équipe (observations, actions à faire, commentaires sur l'université, suivi administratif). Ces notes sont confidentielles et invisibles pour l'étudiant. Elles s'affichent uniquement aux agents et superviseurs de Joda Company.

Procédure pour ajouter une note

  1. Ouvrez le dossier concerné
  2. Localisez la section « Notes internes » (zone de texte libre en bas de page ou dans un onglet dédié)
  3. Cliquez dans la zone pour l'activer
  4. Saisissez votre note (sans limite de caractères, mais restez concis)
  5. Les notes antérieures aparaissent en historique avec timestamps et noms des auteurs
  6. Votre note s'ajoute en bas de l'historique quand vous la saisissez
  7. Cliquez sur « Sauvegarder les notes » ou appuyez sur Ctrl+S
  8. Un message de confirmation « Notes sauvegardées » s'affiche
  9. La note devient immédiatement visible à tous les utilisateurs ayant accès au dossier

Bonnes pratiques pour les notes internes

  • Datez vos entrées : Incluez la date et l'heure (ex : « 23/05/2026 14h30 — »)
  • Soyez précis : Au lieu de « À faire », écrivez « À faire : contacter l'université pour obtenir la lettre d'admission »
  • Utilisez des actions claires :
    • « ✓ FAIT : » pour marquer quelque chose d'accompli
    • « ⚠️ URGENT : » pour signaler une tâche prioritaire
    • « ❓ CLARIFIER : » pour une question en attente de réponse
    • « 📞 APPEL : » pour un suivi téléphonique
  • Évitez les informations sensibles : Ne pas noter les identifiants, numéros de sécurité sociale, etc. (sauf strictement nécessaire pour l'équipe)
  • Soyez professionnel : Ces notes sont des documents de travail internes — adoptez un ton neutre et factuel
  • Mettez à jour régulièrement : Dès qu'il y a un développement, ajoutez une note pour que toute l'équipe soit à jour
  • Référencez les décisions : Notez pourquoi un changement de statut a été fait (ex : « Passé en « Visa en cours » suite à réception de lettre d'admission le 22/05 »)

Exemple de notes bien structurées

Exemple :
23/05/2026 14h30 — ✓ FAIT : Documents reçus validés par Jessica
24/05/2026 09h15 — 📞 APPEL : Université de Pékin demande test HSK supplémentaire — Étudiant en attente de date d'examen
25/05/2026 11h — ⚠️ URGENT : Paiement Tranche 2 en retard de 5 jours — Contacter étudiant demain
25/05/2026 15h30 — ✓ FAIT : Étudiant contacté, paiement prévu pour 26/05

7.3 Enregistrer un nouveau paiement

Le formulaire « Encaisser » permet d'enregistrer manuellement un paiement d'étudiant (reçu en cash, virement bancaire, mobile money, etc.). Les paiements déclarés par l'étudiant via le portail apparaissent automatiquement dans « À valider », pas besoin de passer par ce formulaire.

Procédure détaillée — Onglet « Encaisser » :
  1. Cliquez sur l'onglet « Encaisser » du module Frais
  2. Sélectionnez l'étudiant (champ obligatoire)
    • Tapez le nom ou prénom pour filtrer la liste déroulante
    • Sélectionnez la bonne entrée (affichée sous format « Prénom NOM - Email »)
    • Note : Seuls les étudiants avec au moins un paiement en attente apparaissent
  3. Sélectionnez le type de paiement (champ obligatoire)
    • Bourse (procédure) — Frais de candidature pour la procédure d'admission en université chinoise
    • Mandarin — Inscription ou tranche de cours de mandarin
    • Anglais — Inscription ou tranche de cours d'anglais
    • Inscription — Frais d'inscription administratifs (optionnel)
    • Autre — Paiements divers non catégorisés
  4. Sélectionnez la tranche (champ obligatoire, dépend du type)
    • Pour Procédure : T1 (tranche unique pour procédure standard)
    • Pour Mandarin/Anglais : T1, T2, T3, T4 (4 tranches de cours)
    • Pour Autre : Pas de subdivision, une seule option « Autre »
    • Note : Vous ne pouvez enregistrer qu'une tranche qui n'a pas déjà un paiement validé
  5. Saisissez le montant (champ obligatoire)
    • Format numérique uniquement (pas de caractères spéciaux)
    • Montant en FCFA (devise locale)
    • Montants de référence (voir section 7.7) : Bourse complète = 1 000 000 FCFA, Mandarin T1 = 30 250 FCFA, etc.
    • Vous pouvez saisir un montant différent du référence (paiement partiel, correction, etc.)
    • Maximum accepté : 10 000 000 FCFA (protection contre les erreurs de saisie)
  6. Indiquez la date de paiement (champ obligatoire)
    • Date à laquelle l'étudiant a effectivement payé (jour du virement, remise en cash, etc.)
    • Utilisez le calendrier popup — ne tapez pas manuellement
    • Ne peut pas être une date future
    • Si la date est antérieure à aujourd'hui de plus de 30 jours, un avertissement s'affiche (paiement ancien — à confirmer)
  7. Indiquez la date limite (champ optionnel)
    • Date limite pour le paiement (deadline pour l'étudiant)
    • Utilisée pour calculer les pénalités de retard si applicable
    • Généralement 30 jours après la date de paiement ou selon accord avec l'étudiant
    • Peut être vide si pas de date limite spécifique
  8. Ajoutez une note (optionnel)
    • Zone texte pour contexte ou commentaire : « Reçu en espèces », « Virement banque ABC », etc.
    • Visible uniquement par l'équipe
  9. Vérifiez tous les champs obligatoires (marqués d'une * rouge)
  10. Cliquez sur « Enregistrer » (bouton vert)
  11. Validation serveur effectue les vérifications :
    • Montant > 0 et réaliste
    • Dates cohérentes (paiement ≤ limite)
    • Étudiant + type + tranche ne duplique pas un paiement existant
  12. Une fois validé, un message de succès s'affiche : « Paiement enregistré »
  13. Le paiement apparaît immédiatement dans l'onglet « Validés »
  14. Une écriture comptable (crédit) est créée automatiquement dans le livre comptable
  15. L'étudiant voit le paiement dans son portail (historique des frais)

ErreurCauseSolution

7.4 Valider un paiement déclaré Superviseur

Lorsqu'un étudiant déclare un paiement depuis son portail, il apparaît dans l'onglet « À valider » avec le badge Déclaré par l'étudiant. Seuls les superviseurs peuvent valider ou rejeter ces paiements. Les agents peuvent consulter mais pas agir.

Procédure de validation

  1. Cliquez sur l'onglet « À valider »
  2. Vous voyez une liste des paiements déclarés par des étudiants, triés par date (plus récent en haut)
  3. Pour chaque paiement, vérifiez les informations affichées :
    • Nom de l'étudiant
    • Type de paiement (Bourse, Mandarin, etc.)
    • Montant déclaré
    • Date de déclaration (quand l'étudiant l'a soumis)
    • Statut actuel : Déclaré
  4. Examinez la preuve jointe :
    • Cliquez sur « Voir la preuve » ou « Télécharger » pour ouvrir le justificatif
    • Les preuves acceptées incluent :
      • Capture d'écran de confirmation de virement
      • Reçu mobile money (Orange Money, MTN Money, etc.)
      • Confirmation email de la banque
      • Photo du reçu de paiement (cash conforme, lisible)
    • Vérifiez les points suivants :
      • Montant : Correspond-il au montant déclaré ? (Attention aux frais bancaires qui réduisent le montant reçu)
      • Identité du payeur : Est-ce l'étudiant ou un parent/proche mandaté ?
      • Date du paiement : Est-elle cohérente ? (Ne pas rejeter pour paiement datant de quelques jours)
      • Clarté de la preuve : Est-elle lisible et authentique-looking ?
      • Référence : Y a-t-il une référence de transaction ou numéro de virement ?
  5. Décision :
    • Si conforme : Cliquez sur « Valider »
    • Si incomplet/suspect : Cliquez sur « Rejeter »
  6. Si validation :
    • Ajoutez un commentaire optionnel (ex : « Montant vérifié le 23/05 »)
    • Cliquez sur « Confirmer la validation »
    • Le statut passe immédiatement à Payé
    • Une écriture comptable (entrée) est créée automatiquement
    • Un reçu devient disponible pour téléchargement en PDF
    • L'étudiant reçoit une notification : « Votre paiement a été validé »
  7. Si rejet :
    • Un champ « Raison du rejet » apparaît (obligatoire)
    • Sélectionnez ou saisissez une raison parmi :
      • « Montant ne correspond pas »
      • « Preuve non lisible »
      • « Preuve insuffisante »
      • « Montant partiellement reçu (frais déduits) »
      • « Autre » (libre)
    • Cliquez sur « Confirmer le rejet »
    • Le statut revient à Rejeté
    • L'étudiant reçoit une notification avec la raison du rejet et peut resoumettre
    • Aucune écriture comptable n'est créée

Points de vérification critiques

VérificationÀ chercherAction si absent

Montant exactMontant versé = montant attenduSi différence < 5 % (frais bancaires), accepter. Sinon, rejeter avec raison « Montant partiellement reçu ».

8. Gestion des Universités Superviseur

8.1 Accès au module

8.3 Ajouter une université Superviseur

Seuls les superviseurs peuvent ajouter ou modifier les universités partenaires. Les agents ne peuvent que consulter la liste.

Procédure détaillée :
  1. Depuis le module Universités, cliquez sur le bouton « Nouvelle université » (rouge, en haut à droite)
  2. Un formulaire multi-sections s'ouvre. Remplissez les champs suivants :
  3. Informations générales :
    • Nom de l'université (obligatoire)
      • Nom officiel complet (ex : « Université de Pékin », pas « PKU »)
      • 2-100 caractères, lettres, chiffres et espaces acceptés
      • Doit être unique dans la base (pas de doublons)
    • Pays (obligatoire)
      • Sélectionnez dans la liste déroulante des pays
      • Généralement « Chine » pour les universités partenaires
    • Ville / Province (obligatoire)
      • Texte libre (ex : « Pékin », « Shanghai », « Tianjin »)
      • Utilisé pour filtrer et rechercher
    • Code CEAIE (optionnel)
      • Identifiant officiel de l'université auprès de la Chine (si disponible)
      • Format numérique (ex : 10001 pour PKU)
    • Site web (optionnel)
      • URL officielle de l'université (ex : www.pku.edu.cn)
      • Utile pour vérifier les critères d'admission à jour
  4. Programmes et niveaux :
    • Programmes offerts (obligatoire)
      • Format : Liste séparée par des virgules ou « ; »
      • Exemple : « Master en Génie Informatique, Master en Administration Publique, Licence en Commerce International »
      • Chaque programme sur une ligne si possible (un par ligne = plus lisible)
      • Soyez précis : « Master en Finance » plutôt que « Finance »
      • Vous pouvez en ajouter jusqu'à 50
    • Niveaux d'études acceptés (obligatoire)
      • Cases à cocher pour sélectionner (plusieurs possibles) :
        • ☐ Licence (Bac+3)
        • ☐ Master (Bac+5)
        • ☐ Doctorat (Bac+8)
        • ☐ Formation professionnelle (court terme)
      • Au moins un niveau doit être sélectionné
  5. Critères d'admission :
    • Niveau de langue requis (optionnel)
      • Sélectionnez : Pas requis, HSK 4, HSK 5, HSK 6, TOEFL 80+, IELTS 6.0+, Autre
      • Utilisé pour orienter les candidatures
    • Moyenne minimale requise (optionnel)
      • Format numérique (ex : 3.0 sur 4.0, ou 70 sur 100)
      • Indiquez l'échelle si nécessaire (« 3.0/4.0 »)
    • Autres critères (optionnel)
      • Zone de texte libre pour critères spécifiques (ex : « Expérience professionnelle 2+ ans », « Tests d'entrée requis », « Entretien obligatoire »)
      • Jusqu'à 500 caractères
  6. Paramètres opérationnels :
    • Statut de l'université (obligatoire)
      • ☑ Actif — Les agents peuvent créer des candidatures pour cette université
      • ☐ Inactif — L'université n'apparaît plus dans les listes. Utilisez ce statut pour les universités avec lesquelles vous ne travaillez plus temporairement.
    • Contact administratif (optionnel)
      • Email ou personne de référence pour vos communications avec l'université
      • Utile pour suivre les dossiers
    • Notes internes (optionnel)
      • Informations de contexte pour votre équipe : « Partenaire depuis 2020 », « Haut taux d'acceptation (80%) », etc.
  7. Vérifiez que tous les champs obligatoires (marqués d'une * rouge) sont complétés
  8. Cliquez sur « Enregistrer » (bouton vert)
  9. L'application valide :
    • Unicité du nom (pas d'université avec le même nom)
    • Au moins un programme et un niveau sélectionnés
    • Statut défini
  10. Un message de confirmation s'affiche : « Université créée »
  11. L'université apparaît immédiatement dans la liste et est disponible pour les candidatures

13. Communication — Messages & SMS

13.1 Accès au module

Menu latéral → Communication (ou Messages & SMS). Accessible aux agents, superviseurs et administrateurs.

13.2 Présentation

Le module Communication permet d'envoyer des emails et des SMS aux étudiants, individuellement ou en masse (à plusieurs étudiants en une seule opération). Il comprend deux onglets :

OngletFonctionnalité
MessagesEnvoi d'emails individuels ou groupés. Un SMS est envoyé en parallèle si le numéro de l'étudiant est disponible.
SMSEnvoi de SMS uniquement, avec affichage du solde de crédits SMS disponibles.

13.3 Envoyer un message (email + SMS)

Onglet « Messages » :
  1. Ouvrez le module Communication, onglet « Messages »
  2. La liste de tous les étudiants s'affiche avec une case à cocher devant chacun
  3. Sélectionnez les destinataires :
    • Cochez un ou plusieurs étudiants individuellement
    • Ou utilisez « Tout sélectionner » pour un envoi de masse
  4. Saisissez le sujet du message (objet de l'email)
  5. Saisissez le corps du message (texte à envoyer)
  6. Un compteur de caractères indique la longueur — utile car les SMS sont limités à 160 caractères
  7. Cliquez sur « Envoyer »
  8. L'application envoie :
    • Un email HTML (responsive, avec en-tête Joda Company) à chaque destinataire
    • Un SMS simultané à chaque étudiant disposant d'un numéro de téléphone enregistré
  9. Un message de confirmation indique le nombre d'envois réussis

13.4 Envoyer un SMS

Onglet « SMS » :
  1. Ouvrez l'onglet « SMS »
  2. Le solde de crédits SMS disponibles s'affiche en haut (rechargé par l'administration)
  3. La liste des étudiants disposant d'un numéro de téléphone s'affiche
  4. Sélectionnez les destinataires (une sélection ou tous)
  5. Saisissez le texte du SMS (maximum 160 caractères pour un SMS simple)
  6. Surveillez le compteur de caractères
  7. Cliquez sur « Envoyer le SMS »
  8. Les numéros camerounais sont automatiquement normalisés au format international (237XXXXXXXXX)
  9. Le solde de crédits est décrémenté selon le nombre d'envois
Conseils pour les SMS :
  • Restez sous 160 caractères pour éviter qu'un message soit facturé comme plusieurs SMS
  • Vérifiez le solde de crédits avant un envoi de masse
  • Les SMS ne fonctionnent qu'avec des numéros valides et bien renseignés dans la fiche étudiant
Attention : Les envois de masse sont irréversibles. Vérifiez bien le contenu et la liste des destinataires avant d'envoyer. Un message mal rédigé envoyé à tous les étudiants ne peut pas être rappelé.

13.5 Bonnes pratiques de communication

  • Personnalisez quand c'est possible : Pour un message important à un seul étudiant, préférez l'envoi individuel
  • Soyez concis : Surtout pour les SMS (coût + limite de caractères)
  • Vérifiez les destinataires : Un envoi de masse touche tous les étudiants cochés
  • Privilégiez l'email pour le contenu long : Le SMS pour les rappels courts (échéance, document manquant)
  • Évitez le spam : N'envoyez que des informations utiles pour préserver la confiance des étudiants

14. Workflows Courants

14.1 Onboarding d'un nouvel étudiant (workflow complet)

  1. Créer l'étudiant (module Étudiants → Nouvel étudiant)
  2. Communiquer les identifiants de connexion au portail étudiant
  3. Créer la candidature (module Candidatures → Nouvelle candidature) avec l'université et le programme souhaités
  4. Si cours de langue : inscrire l'étudiant aux cours (module Étudiants → vue détail → Cours)
  5. Créer les paiements attendus (module Frais → Encaisser) selon la procédure choisie
  6. Demander les documents à l'étudiant (il les soumet via le portail)
  7. Valider les documents reçus (module Dossiers → Documents)
  8. Mettre à jour le statut du dossier au fur et à mesure

14.2 Traitement d'un paiement étudiant

  1. Notification reçue : un étudiant a déclaré un paiement
  2. Aller dans Frais → onglet « À valider »
  3. Trouver le paiement marqué « Déclaré par l'étudiant »
  4. Cliquer sur « Voir la preuve » pour vérifier le justificatif
  5. Si conforme → « Valider » le paiement
  6. Le statut passe à « Payé », l'écriture comptable est créée automatiquement
  7. Un reçu est disponible pour téléchargement

14.3 Gestion d'un document non conforme

  1. Ouvrir le dossier concerné
  2. Dans la liste des documents, identifier le document problématique
  3. Cliquer sur « Rejeter » Superviseur
  4. Saisir la raison du rejet (ex : « Image floue, merci de rescanner »)
  5. L'étudiant reçoit une notification et voit la raison dans son portail
  6. Il soumet un nouveau document — le statut repasse à « En attente »
  7. Valider le nouveau document une fois conforme

14.4 Passer un dossier en admission validée

  1. Ouvrir le dossier de l'étudiant
  2. Vérifier que tous les documents requis sont validés
  3. Changer le statut vers « Admission validée » Superviseur
  4. Ajouter une note interne si nécessaire (ex : « Lettre d'admission reçue le JJ/MM »)
  5. L'étudiant voit automatiquement toutes les étapes précédentes cochées dans sa timeline
  6. Engager ensuite la procédure de visa : passer le statut à « Visa en cours »
