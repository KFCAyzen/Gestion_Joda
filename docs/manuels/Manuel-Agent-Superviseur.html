1. Introduction — Rôles et Permissions

1.1 Présentation

Ce manuel est destiné aux agents et superviseurs de Joda Company. Il couvre l'ensemble des opérations quotidiennes : gestion des étudiants, des candidatures, des dossiers, des paiements, et plus encore.

1.2 Différence entre Agent et Superviseur

Les permissions varient selon votre rôle. Les fonctionnalités marquées Superviseur ne sont accessibles qu'aux superviseurs et niveaux supérieurs.

ModuleAgentSuperviseur
Tableau de bordLectureLecture
ÉtudiantsVoir, Créer, ModifierVoir, Créer, Modifier, Supprimer
CandidaturesVoir, Créer, ModifierVoir, Créer, Modifier, Supprimer
DossiersVoir, Créer, ModifierVoir, Créer, Modifier, Supprimer, Valider
UniversitésLecture seuleVoir, Créer, Modifier
PaiementsVoir, Créer, ModifierVoir, Créer, Modifier, Valider
ComptabilitéNon accessibleConsultation
UtilisateursNon accessibleNon accessible
Logs d'activitéNon accessibleNon accessible
PerformancesLectureLecture

1.3 Hiérarchie des rôles

La hiérarchie complète dans l'application (du plus faible au plus fort) :

  1. Étudiant (student) — portail dédié uniquement
  2. Utilisateur (user) — accès lecture basique
  3. Agent — opérations courantes
  4. Superviseur — opérations + validations
  5. Admin — gestion complète
  6. Super Admin — accès total

2. Connexion et Navigation Générale

2.1 Connexion

Étapes de connexion :
  1. Ouvrez l'application Joda Company (navigateur ou application desktop)
  2. Saisissez votre nom d'utilisateur
  3. Saisissez votre mot de passe
  4. Cliquez sur « Se connecter »
  5. Vous arrivez automatiquement sur le tableau de bord

3. Tableau de Bord

3.1 Présentation

Le tableau de bord est votre vue d'ensemble opérationnelle. Il se rafraîchit automatiquement et affiche les données en temps réel.

Le journal central liste toutes les actions récentes de l'équipe. Vous pouvez :

  • Rechercher par description ou nom d'utilisateur
  • Voir les alertes critiques (fond rouge) : paiements à valider, admissions, etc.

4. Gestion des Étudiants

4.1 Accès au module

Menu latéral → Étudiants

4.2 Vue liste des étudiants

La page affiche la liste de tous les étudiants avec :

  • Nom, prénom
  • Email de contact
  • Téléphone
  • Date d'inscription
  • Boutons d'actions

Utilisez la barre de recherche (par nom/prénom/email) ou le filtre par sexe pour retrouver rapidement un étudiant. La liste est paginée (20 étudiants par page).

4.3 Créer un nouvel étudiant

Étapes :
  1. Cliquez sur « Nouvel étudiant » (bouton en haut à droite)
  2. Remplissez les informations d'identité :
    • Prénom et nom (obligatoires)
    • Email de contact personnel (obligatoire)
    • Téléphone
    • Date de naissance, sexe, nationalité
  3. Remplissez le profil académique :
    • Niveau d'études actuel
    • Filière / spécialité
    • Niveau de langue (HSK, TOEFL, etc.)
    • Diplôme obtenu
  4. Sélectionnez le type de programme (procédure seule, procédure + cours, cours seuls)
  5. Cliquez sur « Enregistrer »
  6. Un compte étudiant est automatiquement créé (email interne + mot de passe temporaire)

Informations générées automatiquement à la création :

ChampValeur générée
Email systèmeprenom.nom@students.joda.app (pour la connexion)
Nom d'utilisateurInitiale prénom + début du nom (ex : JDupont)
Mot de passeTemporaire aléatoire (l'étudiant devra le changer à la 1ère connexion)
Note : Communiquez les identifiants de connexion (nom d'utilisateur + mot de passe temporaire) à l'étudiant de façon sécurisée.

4.4 Modifier un étudiant

  1. Dans la liste, cliquez sur l'icône crayon (modifier) de l'étudiant
  2. Modifiez les champs souhaités
  3. Cliquez sur « Enregistrer »

4.5 Voir les détails d'un étudiant

Cliquez sur l'icône œil (voir) pour accéder à la vue détaillée de l'étudiant, qui comprend :

  • Ses informations personnelles complètes
  • Ses candidatures en cours
  • Ses paiements et frais
  • Ses documents soumis

4.6 Inscrire à des cours de langue

  1. Dans la vue détaillée de l'étudiant, accédez à l'onglet « Cours »
  2. Cliquez sur « Inscrire au Mandarin » ou « Inscrire à l'Anglais »
  3. L'inscription crée automatiquement 4 tranches de paiement dans le module Frais

4.7 Télécharger la fiche étudiant

Le bouton « Imprimer / PDF » génère une fiche étudiant complète au format PDF (avec logo Joda Company, informations personnelles, photo si disponible).

4.8 Supprimer un étudiant Superviseur

Action irréversible : La suppression d'un étudiant efface toutes ses données, ses candidatures, ses documents et ses paiements. Cette action déclenche une confirmation explicite avant exécution.

5. Gestion des Candidatures

5.1 Accès au module

Menu latéral → Candidatures

5.2 Vue liste

La page affiche toutes les candidatures avec filtres par statut et université. Recherchez par nom d'étudiant, université ou programme. La liste est paginée (10 candidatures par page).

5.3 Créer une candidature

Étapes :
  1. Cliquez sur « Nouvelle candidature »
  2. Sélectionnez l'étudiant concerné (liste déroulante)
  3. Sélectionnez l'université cible
  4. Saisissez le programme souhaité (ex : « Master en Génie Informatique »)
  5. Sélectionnez le niveau d'études (Licence, Master, Doctorat, etc.)
  6. Indiquez le niveau de langue de l'étudiant
  7. Sélectionnez le type de bourse (complète, partielle)
  8. Cliquez sur « Enregistrer »
Conseil : Une candidature crée automatiquement un dossier dans le module « Dossiers ». Les deux sont liés.

5.4 Statuts des candidatures

StatutDescriptionQui peut modifier
Documents reçusDocuments initiaux reçus de l'étudiantAgent, Superviseur

Admission rejetéeL'université a refuséSuperviseur, Admin

5.5 Modifier une candidature

  1. Cliquez sur l'icône crayon de la candidature
  2. Modifiez les champs souhaités (université, programme, niveau, type de bourse)
  3. Cliquez sur « Enregistrer »
  4. La modification est enregistrée dans les logs d'activité

6. Gestion des Dossiers

6.1 Accès au module

Menu latéral → Dossiers

6.2 Présentation

Le module Dossiers est le cœur de la gestion opérationnelle. Chaque dossier correspond à une candidature et contient :

  • Les informations détaillées de la candidature
  • Les documents soumis par l'étudiant
  • Les notes internes (visibles uniquement par l'équipe)
  • L'historique des changements de statut

6.3 Filtrage et recherche

  • Filtre par statut (menu déroulant)
  • Recherche par nom étudiant, université ou programme
  • Pagination automatique

6.4 Ouvrir un dossier

Cliquez sur un dossier dans la liste pour ouvrir sa vue détaillée. Vous y trouverez :

  • Informations de l'étudiant et de la candidature
  • Statut actuel et menu de changement de statut
  • Zone de notes internes
  • Liste des documents

6.5 Changer le statut d'un dossier

  1. Ouvrez le dossier
  2. Cliquez sur le menu déroulant « Statut »
  3. Sélectionnez le nouveau statut
  4. Cliquez sur « Enregistrer »
  5. Le changement est automatiquement logué et l'étudiant est notifié

6.6 Notes internes

Le champ « Notes internes » permet de laisser des informations pour l'équipe (observations, actions à faire, commentaires sur l'université). Ces notes sont invisibles pour l'étudiant.

  1. Saisissez votre note dans le champ « Notes internes »
  2. Cliquez sur « Sauvegarder les notes »

6.7 Gestion des documents d'un dossier

La section documents du dossier liste tous les documents soumis par l'étudiant :

ActionDescriptionRôle requis
Voir le documentOuvre/télécharge le fichierAgent
ValiderMarque le document comme conformeSuperviseur
RejeterMarque le document non conforme + saisir raisonSuperviseur
Superviseur uniquement : La validation et le rejet de documents nécessitent le rôle superviseur. En tant qu'agent, vous pouvez consulter les documents mais pas les valider.

6.8 Uploader un document depuis le dossier

  1. Dans la vue dossier, section documents
  2. Cliquez sur « Ajouter un document »
  3. Sélectionnez le type de document
  4. Choisissez le fichier (max 10 MB)
  5. Cliquez sur « Envoyer »

7.3 Enregistrer un nouveau paiement

Onglet « Encaisser » :
  1. Sélectionnez l'étudiant dans la liste déroulante
  2. Sélectionnez le type de paiement :
    • Bourse (procédure)
    • Mandarin
    • Anglais
    • Inscription
    • Autre
  3. Sélectionnez la tranche (T1, T2, T3, T4 selon le type)
  4. Saisissez le montant en FCFA
  5. Indiquez la date de paiement
  6. Ajoutez une date limite si applicable
  7. Cliquez sur « Enregistrer »
  8. Une écriture comptable est automatiquement créée

7.4 Valider un paiement déclaré Superviseur

Lorsqu'un étudiant déclare un paiement depuis son portail, il apparaît dans l'onglet « À valider » avec le badge « Déclaré par l'étudiant ».

  1. Allez dans l'onglet « À valider »
  2. Trouvez le paiement concerné
  3. Si l'étudiant a joint une preuve, cliquez sur « Voir la preuve » pour la vérifier
  4. Cliquez sur « Valider » pour confirmer le paiement
  5. Ou « Rejeter » si la preuve est insuffisante
  6. Une fois validé, le statut passe à « Payé » et un reçu est disponible

7.5 Pénalités de retard

Les pénalités sont calculées automatiquement pour les paiements en retard. Elles s'affichent sur la ligne du paiement en retard. L'application synchronise les pénalités au démarrage et peut les recalculer à la demande.

TypeTaux journalier
Procédure bourse10 FCFA/jour
Cours (inscription)500 FCFA/jour
Cours (tranche)1 000 FCFA/jour

7.6 Générer un reçu de paiement

  1. Sur la ligne d'un paiement validé, cliquez sur « Reçu »
  2. Choisissez le format : PDF (téléchargement) ou Word
  3. Le reçu inclut le logo Joda Company, les coordonnées de l'étudiant et les détails du paiement

12. Workflows Courants

12.1 Onboarding d'un nouvel étudiant (workflow complet)

  1. Créer l'étudiant (module Étudiants → Nouvel étudiant)
  2. Communiquer les identifiants de connexion au portail étudiant
  3. Créer la candidature (module Candidatures → Nouvelle candidature) avec l'université et le programme souhaités
  4. Si cours de langue : inscrire l'étudiant aux cours (module Étudiants → vue détail → Cours)
  5. Créer les paiements attendus (module Frais → Encaisser) selon la procédure choisie
  6. Demander les documents à l'étudiant (il les soumet via le portail)
  7. Valider les documents reçus (module Dossiers → Documents)
  8. Mettre à jour le statut du dossier au fur et à mesure

12.2 Traitement d'un paiement étudiant

  1. Notification reçue : un étudiant a déclaré un paiement
  2. Aller dans Frais → onglet « À valider »
  3. Trouver le paiement marqué « Déclaré par l'étudiant »
  4. Cliquer sur « Voir la preuve » pour vérifier le justificatif
  5. Si conforme → « Valider » le paiement
  6. Le statut passe à « Payé », l'écriture comptable est créée automatiquement
  7. Un reçu est disponible pour téléchargement

12.3 Gestion d'un document non conforme

  1. Ouvrir le dossier concerné
  2. Dans la liste des documents, identifier le document problématique
  3. Cliquer sur « Rejeter » Superviseur
  4. Saisir la raison du rejet (ex : « Image floue, merci de rescanner »)
  5. L'étudiant reçoit une notification et voit la raison dans son portail
  6. Il soumet un nouveau document — le statut repasse à « En attente »
  7. Valider le nouveau document une fois conforme

12.4 Passer un dossier en admission validée

  1. Ouvrir le dossier de l'étudiant
  2. Vérifier que tous les documents requis sont validés
  3. Changer le statut vers « Admission validée » Superviseur
  4. Ajouter une note interne si nécessaire (ex : « Lettre d'admission reçue le JJ/MM »)
  5. L'étudiant voit automatiquement toutes les étapes précédentes cochées dans sa timeline
  6. Engager ensuite la procédure de visa : passer le statut à « Visa en cours »
