1. Introduction — Rôles Admin et Super Admin

1.1 Présentation

Ce manuel est destiné aux Administrateurs et Super Administrateurs de Joda Company. En plus de toutes les fonctionnalités accessibles aux agents et superviseurs, vous avez accès à la gestion des utilisateurs, la comptabilité complète, les logs d'activité et la surveillance du système.

1.2 Hiérarchie des rôles

Utilisateur1Lecture basique

Super Admin5Accès total sans restriction

2. Connexion et Navigation

2.1 Connexion

Accès Web (navigateur)

Étapes de connexion — Version web :
  1. Ouvrez un navigateur web sécurisé (Chrome, Firefox, Edge, Safari)
  2. Naviguez vers l'adresse officielle : https://admin.joda.company/login
  3. Vérifiez le certificat SSL (icône cadenas vert dans la barre d'adresse)
  4. Entrez votre nom d'utilisateur (format : InitialePrénom + Nom, ex : JDupont)
  5. Entrez votre mot de passe de façon sécurisée (clavier virtuel recommandé pour les environnements sensibles)
  6. Cochez « Rester connecté » si souhaité (à éviter sur ordinateurs partagés)
  7. Cliquez sur « Se connecter »
  8. Après vérification des identifiants et permissions, vous accédez au tableau de bord administrateur
  9. Votre nom et rôle (Admin/Super Admin) s'affichent en haut à droite
  10. Un badge Super Admin ou Admin indique votre niveau de permission exact

Accès Desktop (application installée)

Étapes de connexion — Version desktop :
  1. Double-cliquez sur l'icône Joda Company Admin sur le bureau ou le menu Démarrage
  2. Attendez le chargement initial de l'application (5-10 secondes, plus lent que le web)
  3. Une fenêtre dédiée s'ouvre avec le formulaire de connexion sécurisé
  4. Saisissez votre nom d'utilisateur
  5. Saisissez votre mot de passe
  6. Cliquez sur « Se connecter »
  7. L'application effectue une synchronisation avec la base de données cloud Supabase (barre de progression visible)
  8. L'application crée également une copie locale de vos données administratives en SQLite pour fonctionnement offline
  9. Vous accédez au tableau de bord administrateur avec toutes les fonctionnalités disponibles
  10. Avantage offline : Si vous perdez la connexion Internet, vous pouvez continuer à consulter les données en local (lecture seule pour certains modules). Les modifications administratives seront synchronisées au retour de la connexion.

Points de sécurité importants pour l'administrateur

Sécurité critique : En tant qu'administrateur, vos identifiants donnent accès à des données sensibles (tous les dossiers étudiants, la comptabilité, les logs d'activité). Respectez ces règles :
  • Ne cliquez jamais sur un lien de connexion envoyé par email — tapez toujours l'URL directement
  • Utilisez un mot de passe fort (>12 caractères, mélange de majuscules, minuscules, chiffres, caractères spéciaux)
  • Changez votre mot de passe tous les 3 mois
  • Ne vous connectez jamais sur des réseaux WiFi publics — utilisez un VPN si nécessaire
  • Déconnectez-vous toujours après chaque session, surtout sur ordinateurs partagés

Tableau des problèmes courants et solutions

ProblèmeCause probableSolution
« Identifiants invalides »Nom d'utilisateur ou mot de passe incorrect (respect de la casse)Vérifiez le nom d'utilisateur exact. Utilisez « Mot de passe oublié » si nécessaire. Vérifiez que votre compte n'a pas été désactivé.
« Certificat SSL invalide » (navigateur)Problème de sécurité ou cache navigateur corrompuATTENTION : Ne contournez jamais cet avertissement. Videz le cache, essayez en mode navigation privée, réessayez.

Conseil : Pour les accès fréquents et hautement sécurisés, utilisez l'application web (plus rapide). L'application desktop est idéale pour les environnements avec connexion Internet instable ou pour les audits offline.

4. Gestion des Étudiants

En tant qu'admin, vous avez accès à toutes les fonctionnalités de l'agent et du superviseur, plus :

  • Supprimer un étudiant (suppression complète : dossier, paiements, compte)
  • Accès complet à toutes les fiches sans restriction
Attention : La suppression d'un étudiant est irréversible. Elle efface :
  • Son compte utilisateur (accès portail)
  • Ses candidatures et dossiers
  • Ses paiements et écritures comptables liées
  • Ses documents uploadés
Une confirmation explicite est demandée avant suppression.

Pour toutes les autres fonctionnalités (création, modification, inscription cours, reçus), référez-vous au Manuel Agent & Superviseur — Section 4.

9.4 Sorties comptables (Dépenses)

Les sorties sont créées manuellement pour enregistrer les dépenses de l'entreprise.

Créer une sortie (dépense)

Procédure détaillée :
  1. Naviguez vers le module Comptabilité depuis le menu latéral
  2. Cliquez sur le bouton « Nouvelle dépense » (en haut à droite, rouge)
  3. Un formulaire multi-champs s'ouvre. Remplissez obligatoirement :
  4. Catégorie (champ obligatoire)
    • Loyer — Loyer bureau + charges mensuelles
    • Salaires — Rémunération équipe + cotisations patron
    • Fonctionnement — Électricité, eau, Internet, téléphone
    • Matériels — Équipement informatique, furniture, maintenance
    • Fournitures — Fournitures bureau, papier, cartouches encre
    • Transports — Déplacements agents, carburant, taxi
    • Communication — Publicité, marketing, événements
    • Partenaires — Paiements aux partenaires, commissions
    • Divers — Dépenses non catégorisables ailleurs
  5. Montant (champ obligatoire)
    • Format : numérique uniquement, sans symbole de devise
    • Montant en FCFA
    • Maximum accepté : 50 000 000 FCFA (protection contre les erreurs)
    • Exemple : saisissez « 1500000 » pour 1 500 000 FCFA
    • Montant doit être > 0
  6. Date de la dépense (champ obligatoire)
    • Date à laquelle la dépense a été effectuée (date du paiement ou de la facture)
    • Utilisez le calendrier popup (ne tapez pas manuellement)
    • Ne peut pas être une date future
    • Si date antérieure de plus de 3 mois, un avertissement s'affiche (dépense ancienne — à confirmer)
  7. Description (champ obligatoire)
    • Texte clair et descriptif (ex : « Facture électricité mai 2026 » plutôt que « Électricité »)
    • 2-200 caractères
    • Incluez si possible : date facture, numéro facture, bénéficiaire/fournisseur
    • Exemple : « Facture téléphonie Orange #INV-2026-05-1234 »
  8. Justificatif / Pièce jointe (champ optionnel mais RECOMMANDÉ)
    • Upload la facture, le bon de caisse, le reçu, ou tout document justificatif
    • Formats acceptés : PDF, JPG, PNG, GIF (max 10 MB par fichier)
    • Cliquez sur « Sélectionner un fichier » ou glissez-déposez le fichier
    • Vérifiez que le fichier est lisible et montre clairement :
      • Montant de la facture
      • Date de la facture
      • Identité du fournisseur
      • Numéro de facture (si disponible)
    • Les justificatifs manquants sont un risque audit — un admin ultra-strict peut les demander ultérieurement
  9. Notes internes (optionnel)
    • Contexte additionnel pour l'équipe (ex : « Approuvé par direction le 20/05 »)
    • Raison si montant ou date inhabituel
  10. Vérifiez tous les champs obligatoires (marqués d'un * rouge)
  11. Cliquez sur « Enregistrer » (bouton vert)
  12. L'application effectue une validation :
    • Montant > 0 et ≤ 50 000 000 FCFA
    • Date révolue
    • Catégorie valide
  13. Un message de succès s'affiche : « Dépense créée »
  14. La dépense apparaît immédiatement dans le livre comptable avec :
    • Fond rouge clair (non validée)
    • Statut : Non validée
    • Votre nom en tant que créateur

Exemples de dépenses bien complétées

Exemple 1 — Dépense courante :
Catégorie : Fonctionnement
Montant : 125000
Date : 20/05/2026
Description : Facture électricité Eneo mai 2026 #1234567
Justificatif : [Facture PDF uploadée]

Exemple 2 — Dépense salariale :
Catégorie : Salaires
Montant : 2500000
Date : 30/04/2026 (fin de mois)
Description : Paie équipe avril 2026 (6 agents + 2 superviseurs)
Justificatif : [Bordereau paie/relevé bancaire]
Notes : Prime de performance agent Dupont incluse

Exemple 3 — Dépense d'investissement :
Catégorie : Matériels
Montant : 850000
Date : 15/05/2026
Description : Achat serveur informatique SSD 1TB (Fnac) #FAC-2026-0512345
Justificatif : [Facture magasin]
Notes : Approbation direction avant 20/05

Erreurs courantes lors de la création

ErreurCauseSolution
« Montant invalide »Montant ≤ 0, format incorrect, ou > 50 000 000 FCFASaisissez un montant numérique positif réaliste, sans espace ni symbole.
« Description vide ou trop courte »Moins de 2 caractèresComplétez la description avec au moins quelques mots explicatifs.
« Date future »Vous avez saisi une date postérieure à aujourd'huiUtilisez le calendrier pour sélectionner une date passée ou aujourd'hui.
« Fichier trop volumineux »Justificatif > 10 MBComprimez l'image ou le PDF, ou scannez en résolution plus basse.
« Format de fichier non accepté »Vous avez tenté d'uploader un fichier non-image/PDF (ex : .docx, .xlsx)Convertissez en PDF ou JPG d'abord, puis réessayez.

Valider une sortie

  1. Trouvez la dépense dans le livre comptable (fond rouge = non validée)
  2. Vérifiez la description et le justificatif si disponible
  3. Cliquez sur « Valider »
  4. La dépense est marquée comme validée avec votre nom et la date
  5. Elle apparaît grisée dans le livre (validée)

9.9 Budgets prévisionnels

L'onglet Budgets permet de définir des budgets prévisionnels par catégorie et par période, puis de suivre les dépenses réelles par rapport à ces prévisions.

Créer un budget :
  1. Module Comptabilité → onglet « Budgets »
  2. Cliquez sur « Nouveau budget »
  3. Sélectionnez la catégorie concernée (ex : Loyer, Salaires, Communication)
  4. Définissez la période (mois, trimestre, année)
  5. Saisissez le montant prévu (plafond budgétaire) en FCFA
  6. Cliquez sur « Enregistrer »

Suivi budgétaire

Une fois un budget créé, le module compare automatiquement les dépenses réelles de la catégorie aux prévisions :

IndicateurSignification
Montant prévuLe plafond budgétaire défini
Montant consomméTotal des sorties validées de la catégorie sur la période
Reste disponiblePrévu − Consommé (alerte si négatif = dépassement)
Pourcentage consomméJauge de progression (vert < 80%, orange 80-100%, rouge > 100%)
Conseil : Définissez vos budgets en début de mois ou d'année. Consultez l'onglet Budgets régulièrement pour anticiper les dépassements avant qu'ils ne surviennent.

10.3 Créer un utilisateur

Procédure détaillée :
  1. Naviguez vers le module Utilisateurs depuis le menu latéral
  2. Cliquez sur le bouton « Nouvel utilisateur » (rouge, en haut à droite)
  3. Un formulaire s'ouvre. Remplissez les champs :
  4. Prénom (obligatoire)
    • 2-50 caractères, lettres uniquement
    • Exemple : « Jean »
  5. Nom de famille (obligatoire)
    • 2-50 caractères, lettres et tirets acceptés
    • Exemple : « Dupont-Martin »
  6. Nom d'utilisateur (obligatoire)
    • Format standard : InitialePrénom + Nom (ex : JDupont)
    • 4-20 caractères, lettres et chiffres uniquement (pas d'espaces ni caractères spéciaux)
    • Doit être unique dans la base — le système vérifie et alerte en cas de doublon
    • Suggestions :
      • Prénom simple + Nom (ex : JeanDupont)
      • Initiales (ex : JD01, JD02 si plusieurs JD)
    • Éventuellement des chiffres pour distinction (ex : JDupont01, JDupont02)
  7. Email professionnel (obligatoire)
    • Format valide : prenom.nom@joda.company ou prenom.nom@societe.cm (à adapter)
    • Doit être unique dans le système
    • Utilisé pour :
      • Réinitialisation du mot de passe (lien envoyé par email)
      • Notifications admin
      • Authentification alternative
    • À éviter : emails génériques (info@, contact@) ou emails personnels (@gmail.com)
  8. Téléphone (optionnel)
    • Format : +237 6XX XXX XXX ou local 6XX XXX XXX
    • Utilisé pour contact direct ou notifications SMS (si activé ultérieurement)
  9. Rôle (obligatoire)
    • Sélectionnez dans la liste déroulante selon la fonction :
      • User — Accès lecture basique uniquement
      • Agent — Opérations courantes (gestion étudiants, dossiers, paiements)
      • Supervisor — Opérations + validations (paiements, documents, statuts)
      • Admin — Gestion complète sauf création/suppression d'autres admins
      • Super Admin — Super Admin uniquement : Accès illimité + création d'autres admins/super_admins
    • Règle de hiérarchie : Vous pouvez créer des utilisateurs de tout rôle inférieur au vôtre
      • Admin peut créer : User, Agent, Supervisor (mais pas Admin ni Super Admin)
      • Super Admin peut créer : tous les rôles y compris Admin et Super Admin
  10. Mot de passe temporaire (obligatoire)
    • Exigences de sécurité :
      • Minimum 8 caractères (recommandé : 12+)
      • Au moins 1 majuscule (A-Z)
      • Au moins 1 minuscule (a-z)
      • Au moins 1 chiffre (0-9)
      • Au moins 1 caractère spécial (!@#$%^&*)
    • Exemple de mot de passe fort : « JoDa#2026!Sec »
    • Évitez :
      • Mots de dictionnaire (password, 12345678)
      • Informations personnelles visibles (nom, date naissance)
      • Mêmes caractères répétés (aaaaaaaa)
    • Flag automatique : « Doit changer le mot de passe » est activé — l'utilisateur sera forcé de le changer à la 1ère connexion
  11. Permissions personnalisées (avancé, optionnel)
    • Par défaut, les permissions sont celles du rôle sélectionné
    • Vous pouvez refinancer celles-ci après la création si des besoins spécifiques existent
    • À laisser par défaut pour la plupart des cas standard
  12. Vérifiez que tous les champs obligatoires (marqués d'un * rouge) sont complétés
  13. Cliquez sur « Créer » (bouton vert)
  14. L'application effectue les validations :
    • Unicité du nom d'utilisateur
    • Unicité de l'email
    • Force du mot de passe
    • Respect de la hiérarchie (vous ne créez pas un rôle ≥ au vôtre)
  15. Un message de succès s'affiche : « Utilisateur créé »
  16. Le compte est créé avec :
    • Statut : Actif
    • Flag : « Doit changer le mot de passe » (forcera changement à 1ère connexion)
  17. Communiquez les identifiants à l'utilisateur de façon SÉCURISÉE (voir section sécurité ci-dessous)

Communication sécurisée des identifiants

IMPORTANT : Ne jamais envoyer les identifiants par email non sécurisé ! Utilisez l'une de ces méthodes :
  • Réunion en personne : Donnez les identifiants verbalement et demandez à l'utilisateur de définir immédiatement un nouveau mot de passe
  • Lien de réinitialisation : Envoyez un email avec lien de réinitialisation (lien valable 24h) — l'utilisateur définit son propre mot de passe
  • Gestionnaire de mots de passe sécurisé : Utilisez Bitwarden, LastPass, ou équivalent si disponible dans votre organisation
  • Message chiffré : Utilisez Signal, WhatsApp, ou communication sécurisée certifiée
Ne transmettez jamais le mot de passe par SMS ou message non chiffré.

10.5 Réinitialiser le mot de passe d'un utilisateur

La réinitialisation de mot de passe est utile quand un utilisateur oublie son mot de passe ou quand vous souhaitez forcer un changement pour des raisons de sécurité.

Procédure côté administrateur

  1. Accédez au module Utilisateurs
  2. Trouvez l'utilisateur dans la liste (utilisez la recherche si nécessaire)
  3. Sur la ligne de cet utilisateur, localisez le bouton « Réinitialiser mot de passe » (ou icône clé)
  4. Cliquez sur « Réinitialiser mot de passe »
  5. Une confirmation s'affiche : « Un email de réinitialisation sera envoyé à [email] »
  6. Confirmez en cliquant sur « Confirmer la réinitialisation »
  7. Un message de succès s'affiche : « Email de réinitialisation envoyé »
  8. L'application envoie automatiquement un email sécurisé à l'adresse du profil de l'utilisateur
  9. Vous pouvez fermer la fenêtre — le processus se poursuit en arrière-plan

Ce qui se passe pour l'utilisateur

Côté utilisateur (flux email) :
  1. L'utilisateur reçoit un email avec le sujet : « Réinitialisation de votre mot de passe Joda Company »
  2. L'email contient un lien sécurisé unique : « https://joda.company/reset-password?token=XXXXX »
  3. Le lien est valable 24 heures uniquement — après expiration, il faudra redemander une réinitialisation
  4. L'utilisateur clique sur le lien (il peut le faire depuis n'importe quel appareil et connexion)
  5. Une page de formulaire s'ouvre : « Définir votre nouveau mot de passe »
  6. L'utilisateur saisit son nouveau mot de passe (mêmes exigences que lors de la création : 8+ caractères, majuscules, minuscules, chiffres, caractères spéciaux)
  7. Il confirme en cliquant « Changer le mot de passe »
  8. Le mot de passe est changé immédiatement — il peut dès lors se reconnecter avec le nouveau
  9. Il reçoit un email de confirmation : « Votre mot de passe a été changé avec succès »

Important à savoir

AspectDétail
Email utiliséL'email du lien est celui enregistré dans le profil utilisateur (champ « Email »)
Validité du lien24 heures après l'envoi — après, l'utilisateur doit redemander une réinitialisation
Nombre de tentativesL'utilisateur peut essayer autant de fois qu'il veut avant le timeout 24h
Compte temporairement bloqué ?Non — l'utilisateur peut toujours tenter de se connecter avec son ancien mot de passe (qui ne fonctionne plus après changement)
Qui change le mot de passeL'utilisateur lui-même via le lien — l'administrateur ne connaît pas le nouveau mot de passe
HistoriqueLe changement est enregistré dans les logs d'activité comme « password_reset »

Cas d'usage courants

  • Utilisateur oublie son mot de passe : Cliquez sur « Réinitialiser mot de passe », l'utilisateur reçoit un email avec lien de changement
  • Renouvellement de sécurité trimestriel : Forcez une réinitialisation pour tous les comptes actifs tous les 3 mois
  • Email de l'utilisateur change : Modifiez d'abord l'adresse email dans le profil, puis réinitialisez — le lien sera envoyé à la nouvelle adresse
  • Utilisateur n'a pas reçu l'email : Vérifiez que l'email est correct dans le profil. Essayez de réinitialiser à nouveau. Vérifiez les spams de l'utilisateur.

Différence avec « Forcer le changement de mot de passe »

Note de sécurité : Si un utilisateur signale un accès non autorisé à son compte, réinitialisez immédiatement son mot de passe, vérifiez les logs de ses dernières actions, et demandez-lui de se reconnecter avec un nouveau mot de passe personnel.

10.6 Activer / Désactiver un compte

Un compte désactivé ne peut plus se connecter à l'application. L'historique et les données sont conservés. Utile pour les départs d'employés sans suppression définitive des données.

  1. Cliquez sur le bouton bascule Actif/Inactif de l'utilisateur
  2. Confirmez l'action dans la boîte de dialogue

10.7 Supprimer un utilisateur

Action irréversible : La suppression efface le compte définitivement. Les données historiques (paiements validés par cet utilisateur, logs) sont conservées mais le compte ne peut pas être restauré. Préférez désactiver le compte plutôt que de le supprimer.

Règles de suppression :

  • Un admin ne peut pas supprimer un autre admin ou un super_admin
  • Un utilisateur ne peut pas supprimer son propre compte
  • Seul un super_admin peut supprimer un compte admin

Procédure de modification des permissions

Matrice d'héritage des permissions

Les permissions granulaires s'ajoutent ou se retirent du rôle de base :

Attention : Les permissions granulaires peuvent créer de la confusion. Lignes directrices :
  • Documentez les raisons des permissions personnalisées (dans les notes du profil utilisateur)
  • Gardez le nombre de cas granulaires au minimum (organisation plus simple = moins de bugs)
  • Révisez régulièrement les permissions granulaires (mensuellement) — elles peuvent devenir obsolètes
  • Évitez de donner des permissions non cohérentes (ex : « Valider paiements » sans « Voir paiements »)
Conseil : Pour la plupart des cas standard (agents, superviseurs, admins classiques), les permissions par défaut du rôle suffisent. Les permissions granulaires sont vraiment utiles pour :
  • Agents spécialisés avec capacités supplémentaires
  • Rôles hybrides temporaires (audit, formation, transition)
  • Restrictions pour raisons de sécurité ou conformité (séparation des tâches)
Ninvoquez-les que si vous avez une raison clairement documentée.

14. Sécurité et Bonnes Pratiques

14.1 Gestion des accès

  • Ne partagez pas les exports PDF de la comptabilité en dehors de l'équipe direction
  • Les logs d'activité ne doivent pas être diffusés aux agents

14.4 Sauvegarde

Important : La base de données est hébergée sur Supabase (cloud). Supabase effectue des sauvegardes automatiques quotidiennes. En cas de problème majeur, contactez le prestataire technique Joda Company.

14.5 Application desktop et mode hors-ligne

L'application desktop synchronise les données entre le cloud Supabase et une base SQLite locale. En cas de coupure Internet :

  • Les données locales restent accessibles
  • Les modifications locales sont synchronisées dès le retour de la connexion
  • Uploads de documents et notifications nécessitent une connexion active

17. Workflows Administratifs Courants

17.1 Création d'un nouveau membre de l'équipe

  1. Menu Utilisateurs → « Nouvel utilisateur »
  2. Remplir : prénom, nom, email professionnel, téléphone

  6. Documenter la correction dans les notes internes du dossier
