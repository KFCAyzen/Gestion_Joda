1. Introduction — Rôles Admin et Super Admin

1.1 Présentation

Ce manuel est destiné aux Administrateurs et Super Administrateurs de Joda Company. En plus de toutes les fonctionnalités accessibles aux agents et superviseurs, vous avez accès à la gestion des utilisateurs, la comptabilité complète, les logs d'activité et la surveillance du système.

1.2 Hiérarchie des rôles

RôleNiveauAccès
Étudiant0Portail étudiant uniquement
Utilisateur1Lecture basique
Agent2Opérations courantes
Superviseur3Validations + suppression
Admin4Gestion complète sauf Super Admin
Super Admin5Accès total sans restriction

1.3 Différences Admin vs Super Admin

FonctionnalitéAdminSuper Admin
Toutes opérations agents/superviseursOuiOui
Comptabilité complète (validation sorties)OuiOui
Gestion des utilisateursOui (sauf super_admin)Oui (tous)
Supprimer un adminNonOui
Créer un admin ou super_adminNonOui
Logs d'activité completsOuiOui
Stockage et monitoringOuiOui
Exporter comptabilité (PDF)OuiOui

4. Gestion des Étudiants

En tant qu'admin, vous avez accès à toutes les fonctionnalités de l'agent et du superviseur, plus :

  • Supprimer un étudiant (suppression complète : dossier, paiements, compte)
  • Accès complet à toutes les fiches sans restriction
Attention : La suppression d'un étudiant est irréversible. Elle efface :
  • Son compte utilisateur (accès portail)
  • Ses candidatures et dossiers
  • Ses paiements et écritures comptables liées
  • Ses documents uploadés
Une confirmation explicite est demandée avant suppression.

Pour toutes les autres fonctionnalités (création, modification, inscription cours, reçus), référez-vous au Manuel Agent & Superviseur — Section 4.

Accès complet : créer, modifier, supprimer, valider toutes les candidatures sans restriction.

Pour les détails opérationnels, référez-vous au Manuel Agent & Superviseur — Section 5.

Pour les détails opérationnels, référez-vous au Manuel Agent & Superviseur — Section 7.

9. Comptabilité — Livre Comptable Complet

9.1 Accès

Menu latéral → Comptabilité

9.2 Vue d'ensemble

Le livre comptable consolide toutes les entrées (revenus) et sorties (dépenses) de Joda Company. En tant qu'admin, vous pouvez créer, modifier, valider et exporter les écritures comptables.

9.3 Entrées comptables (Revenus)

Les entrées sont créées automatiquement lors de la validation d'un paiement étudiant. Vous n'avez généralement pas à les créer manuellement.

Type d'entréeOrigine
paiement_procedureValidation d'un paiement de type bourse/inscription
paiement_coursValidation d'un paiement de type mandarin/anglais
revenus_diversEntrée manuelle pour revenus exceptionnels

9.4 Sorties comptables (Dépenses)

Les sorties sont créées manuellement pour enregistrer les dépenses de l'entreprise.

Créer une sortie (dépense)

  1. Dans le module Comptabilité, cliquez sur « Nouvelle dépense »
  2. Sélectionnez la catégorie :
    • Loyer
    • Salaires
    • Fonctionnement
    • Matériels
    • Fournitures
    • Transports
    • Communication
    • Partenaires
    • Divers
  3. Saisissez le montant en FCFA
  4. Saisissez la date de la dépense
  5. Rédigez une description claire
  6. Optionnellement, uploadez un justificatif (facture, bon de caisse, etc.) — PDF ou image, max 10 MB
  7. Cliquez sur « Enregistrer »
  8. La dépense apparaît en rouge dans le livre avec le statut « Non validée »

Valider une sortie

  1. Trouvez la dépense dans le livre comptable (fond rouge = non validée)
  2. Vérifiez la description et le justificatif si disponible
  3. Cliquez sur « Valider »
  4. La dépense est marquée comme validée avec votre nom et la date
  5. Elle apparaît grisée dans le livre (validée)

9.5 Vue du livre comptable

Le livre affiche un tableau chronologique avec :

ColonneDescription
HeureDate et heure de l'opération
DescriptionNature de l'opération
CatégorieType d'opération
Montant+ (entrée) ou - (sortie) en FCFA
ValidateurNom de l'agent qui a validé (sorties)

9.6 Filtres

  • Par période : sélecteur date de début et date de fin
  • Par type/catégorie : paiements, cours, dépenses diverses, etc.

9.7 Export PDF

Générer un rapport comptable PDF :
  1. Appliquez les filtres souhaités (période, catégorie)
  2. Cliquez sur « Exporter PDF »
  3. Un rapport formaté est généré avec :
    • En-tête Joda Company (logo, coordonnées)
    • Tableau des entrées et sorties
    • Totaux entrées / sorties
    • Balance (entrées - sorties)
    • Date d'export et période couverte
  4. Le fichier se télécharge automatiquement

9.8 Réconciliation quotidienne

Le module affiche en haut la somme des entrées et sorties du jour pour faciliter la réconciliation quotidienne de caisse.

10. Gestion des Utilisateurs et Permissions

10.1 Accès

Menu latéral → Utilisateurs

10.2 Vue liste des utilisateurs

La page affiche tous les comptes utilisateurs avec :

  • Nom, prénom, nom d'utilisateur
  • Email
  • Rôle (badge coloré)
  • Statut actif/inactif
  • Actions disponibles

Utilisez la recherche (par nom/username/email) ou le filtre par rôle pour retrouver un utilisateur.

10.3 Créer un utilisateur

  1. Cliquez sur « Nouvel utilisateur »
  2. Remplissez les champs :
    • Prénom et nom (obligatoires)
    • Nom d'utilisateur — doit être unique dans le système
    • Email — adresse email professionnelle, doit être unique
    • Téléphone
    • Rôle — sélectionnez selon la responsabilité
    • Mot de passe temporaire — minimum 8 caractères
  3. Cliquez sur « Créer »
  4. Le compte est créé avec le flag « Doit changer le mot de passe » — l'utilisateur sera forcé de le changer à la 1ère connexion
  5. Communiquez les identifiants à l'utilisateur de façon sécurisée
Règle de création : Un admin ne peut créer que des comptes de niveau inférieur (agent, supervisor, user, student). Seul un super_admin peut créer un compte admin ou super_admin.

10.4 Modifier un utilisateur

  1. Cliquez sur l'icône crayon de l'utilisateur
  2. Modifiez nom, email, téléphone ou rôle selon besoin
  3. Cliquez sur « Enregistrer »

10.5 Réinitialiser le mot de passe d'un utilisateur

  1. Sur la ligne de l'utilisateur, cliquez sur « Réinitialiser mot de passe »
  2. Un email de réinitialisation est automatiquement envoyé à l'adresse de l'utilisateur
  3. L'utilisateur suit le lien reçu par email pour définir son nouveau mot de passe
Note : La réinitialisation envoie un lien sécurisé (valable 24h) vers l'email enregistré dans le profil de l'utilisateur.

10.6 Activer / Désactiver un compte

Un compte désactivé ne peut plus se connecter à l'application. L'historique et les données sont conservés. Utile pour les départs d'employés sans suppression définitive des données.

  1. Cliquez sur le bouton bascule Actif/Inactif de l'utilisateur
  2. Confirmez l'action dans la boîte de dialogue

10.7 Supprimer un utilisateur

Action irréversible : La suppression efface le compte définitivement. Les données historiques (paiements validés par cet utilisateur, logs) sont conservées mais le compte ne peut pas être restauré. Préférez désactiver le compte plutôt que de le supprimer.

Règles de suppression :

  • Un admin ne peut pas supprimer un autre admin ou un super_admin
  • Un utilisateur ne peut pas supprimer son propre compte
  • Seul un super_admin peut supprimer un compte admin

10.8 Gestion des permissions granulaires

Au-delà des permissions par défaut du rôle, vous pouvez personnaliser les permissions de chaque utilisateur.

Permissions disponibles par module :

ModulePermissions configurables
ÉtudiantsVoir, Créer, Modifier, Supprimer
CandidaturesVoir, Créer, Modifier, Supprimer
DossiersVoir, Créer, Modifier, Supprimer, Valider
UniversitésVoir, Créer, Modifier, Supprimer
PaiementsVoir, Créer, Modifier, Supprimer, Valider
ComptabilitéVoir, Créer, Modifier, Supprimer, Valider, Exporter
UtilisateursVoir, Créer, Modifier, Supprimer, Gérer permissions
RapportsVoir, Exporter

Modifier les permissions d'un utilisateur :

  1. Sur la ligne de l'utilisateur, cliquez sur « Gérer les permissions »
  2. Le panneau des permissions s'ouvre
  3. Les permissions par défaut de son rôle sont indiquées
  4. Cochez ou décochez les permissions personnalisées
  5. Les modifications sont sauvegardées en temps réel (pas de bouton « Enregistrer » séparé)
Usage recommandé : Les permissions granulaires sont utiles pour des cas spéciaux (ex : un agent qui peut valider les documents pour un dossier précis, ou un superviseur qui ne doit pas voir la comptabilité). Pour les cas standard, les permissions par défaut du rôle suffisent.

11. Logs d'Activités — Traçabilité Complète

11.1 Accès

Menu latéral → Logs d'activités

11.2 Présentation

Le module Logs d'activités est votre outil d'audit. Chaque action effectuée dans l'application est enregistrée avec :

  • Qui — utilisateur et rôle
  • Quoi — type d'action et description
  • Quand — date et heure précises
  • Où — entité concernée (étudiant ID, dossier ID, etc.)
  • Métadonnées — ancien et nouveau statut, montants, etc.

11.4 Filtres avancés

  • Type d'activité : sélectionner dans la liste déroulante
  • Utilisateur : filtrer par membre de l'équipe
  • Entité : filtrer par type (étudiant, dossier, paiement, etc.)
  • Période : date de début et date de fin
  • Pagination : 25 logs par page

11.5 Lecture d'un log

InformationLocalisation
Icône coloréeType d'action (créa/modif/suppression/etc.)

11.6 Cas d'usage courants des logs

  • Audit d'un paiement contesté : rechercher par étudiant ID et type « payment_validate »
  • Vérifier qui a changé un statut : rechercher par dossier ID et type « dossier_status_change »
  • Surveiller les connexions : filtrer par type « login »/« logout »
  • Tracer une suppression : filtrer par types « *_delete »

14. Sécurité et Bonnes Pratiques

14.1 Gestion des accès

  • Changez régulièrement votre mot de passe admin (recommandé : tous les 3 mois)
  • Ne communiquez jamais votre mot de passe admin à qui que ce soit
  • Désactivez immédiatement les comptes des employés qui quittent l'entreprise
  • N'accordez que les permissions strictement nécessaires (principe du moindre privilège)

14.2 Surveillance des logs

  • Vérifiez régulièrement les logs de connexion pour détecter des accès inhabituels
  • En cas d'anomalie, désactivez immédiatement le compte concerné
  • Conservez une copie des rapports comptables PDF chaque mois

14.3 Données sensibles

  • Les données étudiants sont confidentielles — limitez l'accès au minimum nécessaire
  • Ne partagez pas les exports PDF de la comptabilité en dehors de l'équipe direction
  • Les logs d'activité ne doivent pas être diffusés aux agents

14.4 Sauvegarde

Important : La base de données est hébergée sur Supabase (cloud). Supabase effectue des sauvegardes automatiques quotidiennes. En cas de problème majeur, contactez le prestataire technique Joda Company.

14.5 Application desktop et mode hors-ligne

L'application desktop synchronise les données entre le cloud Supabase et une base SQLite locale. En cas de coupure Internet :

  • Les données locales restent accessibles
  • Les modifications locales sont synchronisées dès le retour de la connexion
  • Uploads de documents et notifications nécessitent une connexion active

15. Workflows Administratifs Courants

15.1 Création d'un nouveau membre de l'équipe

  1. Menu Utilisateurs → « Nouvel utilisateur »
  2. Remplir : prénom, nom, email professionnel, téléphone
  3. Définir le rôle approprié (agent ou superviseur pour le terrain)
  4. Définir un mot de passe temporaire fort
  5. Créer le compte → le flag « doit changer le mot de passe » est activé
  6. Communiquer les identifiants de façon sécurisée (de vive voix ou message sécurisé)
  7. Si besoin, configurer des permissions personnalisées (Gérer les permissions)
  8. Vérifier dans les logs le premier login de l'employé

15.2 Clôture mensuelle comptable

  1. Module Comptabilité → filtrer sur le mois écoulé
  2. Vérifier que toutes les sorties du mois sont validées (sinon les valider)
  3. Vérifier la cohérence des entrées (liées aux paiements validés)
  4. Cliquer sur « Exporter PDF » pour le rapport du mois
  5. Sauvegarder le rapport dans votre archive mensuelle
  6. Comparer avec les paiements validés dans le module Frais

15.3 Départ d'un employé

  1. Module Utilisateurs → trouver le compte de l'employé
  2. Désactiver immédiatement le compte (le toggle Actif/Inactif)
  3. Si l'employé avait des dossiers en cours, les réassigner à un autre agent
  4. Vérifier dans les logs d'activité les dernières actions de l'employé
  5. Conserver le compte désactivé pour l'historique (ne pas supprimer)

15.4 Audit d'une contestation de paiement

  1. Module Logs d'activités
  2. Filtrer par type « payment_validate » et par étudiant concerné
  3. Identifier le log de validation : qui a validé, quand, quel montant
  4. Si disponible, consulter la preuve de paiement attachée (module Frais → voir la preuve)
  5. Si une erreur est confirmée, corriger via le module Frais (modifier ou annuler le paiement)
  6. Documenter la correction dans les notes internes du dossier
